# gray_to_binary (`gray_to_binary.sv`)

## 개요

그레이 코드(Gray Code)를 이진 코드(Binary Code)로 역변환하는 조합 논리 모듈입니다. `binary_to_gray`의 역함수에 해당하며, CDC FIFO의 포인터 복원, 회전 엔코더 디코딩 등에서 사용됩니다. 그레이 코드로 전송된 포인터를 수신 도메인에서 이진 값으로 되돌릴 때 필수적으로 사용됩니다.

## 블록 다이어그램

```mermaid
graph LR
    A["A[N-1:0]\n(그레이 코드 입력)"] --> XOR_TREE["XOR 트리 연산\nZ[i] = XOR(A[N-1:i])"]
    XOR_TREE --> Z["Z[N-1:0]\n(이진 코드 출력)"]
```

## 포트 목록

| 포트명 | 방향 | 비트폭 | 설명 |
|--------|------|--------|------|
| `A` | input | `[N-1:0]` | 그레이 코드 입력 |
| `Z` | output | `[N-1:0]` | 이진 코드 출력 |

## 파라미터

| 파라미터명 | 기본값 | 설명 |
|-----------|--------|------|
| `N` | `-1` (필수 지정) | 변환할 비트 수 |

## 동작 설명

그레이 코드에서 이진 코드로의 역변환은 각 비트에 대해 상위 비트들의 XOR를 적용합니다:

```
Z[i] = A[N-1] ^ A[N-2] ^ ... ^ A[i]
     = ^A[N-1:i]   (reduction XOR)
```

- `Z[N-1] = A[N-1]`
- `Z[N-2] = A[N-1] ^ A[N-2]`
- `Z[N-3] = A[N-1] ^ A[N-2] ^ A[N-3]`
- ...

**예시 (N=4):**

| 그레이(A) | 이진(Z) |
|---------|---------|
| 0000    | 0000    |
| 0001    | 0001    |
| 0011    | 0010    |
| 0010    | 0011    |
| 0110    | 0100    |

## 내부 구조

`generate for` 루프를 사용하여 N개의 비트 각각에 reduction XOR를 적용합니다:

```systemverilog
for (genvar i = 0; i < N; i++)
    assign Z[i] = ^A[N-1:i];
```

순수 조합 논리이며 클록이나 플립플롭을 사용하지 않습니다.

## 의존성

없음 (독립 모듈)

## 사용 예시

```systemverilog
// 8비트 그레이 코드 → 이진 변환
gray_to_binary #(
    .N ( 8 )
) i_g2b (
    .A ( gray_pointer  ),
    .Z ( bin_pointer   )
);
```
