# binary_to_gray (`binary_to_gray.sv`)

## 개요

이진 코드(Binary Code)를 그레이 코드(Gray Code)로 변환하는 조합 논리 모듈입니다. 그레이 코드는 연속하는 두 값 사이에서 단 1비트만 변하는 특성이 있어, CDC(Clock Domain Crossing) FIFO의 포인터 전송, 회전 엔코더 인터페이스, ADC 등에서 광범위하게 사용됩니다. 특히 멀티비트 신호를 클록 도메인 경계로 넘길 때 메타스테빌리티로 인한 잘못된 값 캡처를 방지하기 위해 활용됩니다.

## 블록 다이어그램

```mermaid
graph LR
    A["A[N-1:0]\n(이진 입력)"] --> XOR["XOR 연산\nZ = A ^ (A >> 1)"]
    XOR --> Z["Z[N-1:0]\n(그레이 코드 출력)"]
```

## 포트 목록

| 포트명 | 방향 | 비트폭 | 설명 |
|--------|------|--------|------|
| `A` | input | `[N-1:0]` | 이진 코드 입력 |
| `Z` | output | `[N-1:0]` | 그레이 코드 출력 |

## 파라미터

| 파라미터명 | 기본값 | 설명 |
|-----------|--------|------|
| `N` | `-1` (필수 지정) | 변환할 비트 수 |

## 동작 설명

그레이 코드 변환은 다음 수식으로 수행됩니다:

```
Z = A ^ (A >> 1)
```

즉, 각 비트 `Z[i]`는 `A[i] XOR A[i+1]`로 계산됩니다 (최상위 비트 `Z[N-1]`은 `A[N-1]`과 동일). 이 연산은 순수 조합 논리로 구현되며 지연 없이 즉시 출력됩니다.

**예시 (N=4):**

| 이진(A) | 그레이(Z) |
|---------|---------|
| 0000    | 0000    |
| 0001    | 0001    |
| 0010    | 0011    |
| 0011    | 0010    |
| 0100    | 0110    |

연속하는 값 사이에서 정확히 1비트만 변화합니다.

## 내부 구조

- 레지스터 없는 순수 조합 논리 (`assign` 문 하나)
- `assign Z = A ^ (A >> 1);` 단일 구문으로 구현

## 의존성

없음 (독립 모듈)

## 사용 예시

```systemverilog
// 8비트 이진 → 그레이 코드 변환
binary_to_gray #(
    .N ( 8 )
) i_b2g (
    .A ( binary_value ),
    .Z ( gray_value   )
);
```
